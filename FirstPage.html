HTML5

HTML (от англ. HyperText Markup Language — «язык гипертекстовой разметки») — стандартизированный язык разметки документов во Всемирной паутине. Большинство веб-страниц содержат описание разметки на языке HTML (или XHTML). Язык HTML интерпретируется браузерами; полученный в результате интерпретации форматированный текст отображается на экране монитора компьютера или мобильного устройства.

  • DOCTYPE — предназначен для указания типа текущего документа — DTD (document type definition, описание типа документа). Это необходимо, чтобы браузер понимал, как следует интерпретировать текущую веб-страницу
  • html — является контейнером, который заключает в себе все содержимое веб-страницы, включая теги head и body.
  • head — предназначен для хранения других элементов, цель которых — помочь браузеру в работе с данными. Также внутри контейнера head находятся метатеги, которые используются для хранения информации предназначенной для браузеров и поисковых систем
  • title — определяет заголовок документа
  • h1 — HTML предлагает шесть заголовков разного уровня, которые показывают относительную важность секции, расположенной после заголовка.
  • div — является блочным элементом и предназначен для выделения фрагмента документа с целью изменения вида содержимого. Как правило, вид блока управляется с помощью стилей. Чтобы не описывать каждый раз стиль внутри тега, можно выделить стиль во внешнюю таблицу стилей
  • p — определяет текстовый абзац
  • ul — устанавливает маркированный список. Каждый элемент списка должен начинаться с тега li.
  • mdash — длинное тире (НЕ ТЕГ)
  • header — используется для хранения заголовка страницы,задает «шапку» сайта или раздела, в которой обычно располагается заголовок.
  • footer — содержит информацио о странице и разделах, задаёт «подвал» сайта или раздела, в нём может располагаться имя автора, дата документа, контактная и правовая информация.
  • nav — задает навигацию по сайту. Если на странице несколько блоков ссылок, то в nav обычно помещают приоритетные ссылки.
  • aside — определяет блок сбоку от контента для размещения рубрик, ссылок на архив, меток и другой информации. Такой блок, как правило, называется «сайдбар» или «боковая панель».
  • section — задаёт раздел документа, может применяться для блока новостей, контактной информации, глав текста, вкладок в диалоговом окне и др. Раздел обычно содержит заголовок.
  • article — задает содержание сайта вроде новости, статьи, записи блога, форума или др.(текст)

CSS

Каскадные таблицы стилей (Cascading Style Sheets = CSS) — это язык, который отвечает за визуальное представление документов пользователю. Под документом мы будем понимать набор информации о структуре страницы, описываемый языком разметки. А представление документа пользователю, в свою очередь, означает его преобразование в удобную для восприятия форму.

  • font-family — определяет шрифт
  • background-image — фоновое изображение (background-image: url("fon.jpg"))
  • color — это цвет текста
  • text-align — выравнеие текста
  • width — ширина (не включает толщину границ вокруг элемента, значение отступов и полей)
  • max-width — устанавливает максимальную ширину элемента
  • height — устанавливает высоту блочных или заменяемых элементов (высота не включает толщину границ вокруг элемента, значение отступов и полей)
  • display —многоцелевое свойство, которое определяет, как элемент должен быть показан в документе.
  • border-radius — определяет радиус кривизны, от 1 до 4 параметров. чтобы не было острых углов
  • background — определяет цвет фона
  • padding — устанавливает значение полей вокруг содержимого элемента
  • border — позволяет одновременно установить толщину, стиль и цвет границы вокруг элемента. Значения могут идти в любом порядке, разделяясь пробелом
  • margin —устанавливает величину отступа от каждого края элемента. Отступом является пространство от границы текущего элемента до внутренней границы его родительского элемента
  • font-size — устанавливает размер элемента
  • flexbox — чтобы убрать недостатки при создании самых разных HTML конструкций, в том числе адаптированных под разную ширину и высоту, и сделать верстку логичной и простой
    • box-ordinal-group — определяет порядок отображения элементов. Элементы с меньшим значением отображаются раньше тех, значения которых больше
    • box-flex — определяет является ли дочерний элемент box гибким или нет гибий элемент может увеличивать или уменьшать размер в зависимости от изменений размеров box элемента. Значение свойства - относительное значение определяющее гибкость вложенного элемента.
    • box-align — определяет как выравниваются дочерние элементы box элемента. start по левому или верхнему краю контейнера:: end по нижнему или по правому краю:: center по центру:: baseline по базовой линии текста (применяется только к горизонтальному контенту):: stretch растягивается так, чтобы не было пустого пространства в блоке
    • box-direction — определяет направление, в котором дочерние элементы будут размещены в box контейнере ( box-direction: reverse; ИЛИ normal )
    • box-pack — определяет как дочерние элементы должны размещаться в box элементе, когда он больше по размеру чем дочерние элементы (start|end|center|justify)
    • box-sizing — Позволяет делать 2 вещи и определять элементы так: content-box - width и height задают ширину и высоту контента и не включают в себя значения отступов, полей и границ ИЛИ border-box - Свойства width и height включают в себя значения полей и границ, но не отступов (margin). (box-sizing: content-box;)
